# bf-109: Multi-directory JSONL support

Feature was fully implemented across v0.9.5–v0.9.7:

- **v0.9.5**: Auto-discovery of all project dirs under `~/.claude/projects/*` (replaced single-project lookup)
- **v0.9.6**: Recursive JSONL scanning to include subagent files (`<project>/<uuid>/subagents/agent-*.jsonl`)
- **v0.9.7**: `--extra-dirs` CLI flag and `CCDASH_EXTRA_DIRS` env var for scanning additional root directories

Implementation lives in `internal/metrics/tokens.go`:
- `buildDefaultProjectsDirs()` — builds the list of root dirs to scan
- `findAllProjectDirs()` — enumerates project dirs across all roots
- `findJSONLFilesRecursive()` — walks each project dir recursively to find all `.jsonl` files
- `TokenCollector.AddProjectsDir()` — runtime method for adding extra roots
- Dashboard wired up via `ui.Dashboard.AddProjectsDirs()` (called from main with `--extra-dirs` values)

All tests pass. Build is clean.
